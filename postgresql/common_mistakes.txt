

Using NOT IN to exclude
such as: select... where.... not in (select..._)
It doesn't work the way you expect

Nulls throw this, but even if there are no nulls - subplan instead of an anti-join - so a problem for query planning
Can be ok if you know there are no nulls eg NOT IN (1,34,5,7)

Instead, should use not exists (select .... ) and col is not null;

Selecting ranges with BETWEEN
eg BETWEEN 1 AND 100

-- it's inclusive, so shouild be explicit using a where

Uppercase identifies

Postgres makes everything lowercase unless you double quote it

eg CREATE TABLE Customers
or CREATE TABLE "Invoices"

using it

TABLE Customers - returns select * from ....

TABLE "Customers" - fails as it is really customers

For pretty column names in reports, use:

select amount as "Reconciliation Amount" ....
